getting started

  • Partner with your developers
  • Choose a framework
  • Fork it on GitHub
  • Set up a hosted version for your whole team
  • Install a local version on your machine
  • Start or join a code review process
  • Merge in new versions of your framework over time
  • need partnership, in it together, roadblocks only they solve
  • choose framework like bootstrap/foundation
  • fork on github, explain
  • version on a hosted site, reference, prototypes
  • local version on your machine
  • local server and builder on your machine like thin, jekyll & rake
  • code reviews, explain
  • process for merging
  • hope I've inspired you
